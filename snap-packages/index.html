3dbbeb1 on Dec 14, 2023

ghvmctl

Release/Commit:3dbbeb1
DateDec 14, 2023

refactor: cleanup ghvmctl and ghvmctl-runner

  • CI: Be more verbose on launched commands

We have various commands launched, it may be hard to find out what did not work on failures, so let’s add more debugging info.

  • ghvmctl: Do not hardcode vm name

It’s supposed to be configurable, but the variable is only used once.

Ensure we always use it.

  • ghvmctl: Do not shift on missing command

Otherwise the usage message won’t ever be shown, because due to set -e we exit earlier than that.

  • ghvmctl: Fix a shellcheck error

  • CI: Run shellcheck on source files

There’s lots of shell scripting involved here, better to ensure it’s all sane.

  • ghvmctl: Make easier to run it as normal binary for testing

Ensure it can be ran also as outside the snap, making it easier to test in a local lxd instance.

  • ghvmctl: Assume yes for apt-get by default

  • ghvmctl: Fail if running a snap fails

We use some wait time for this, we may provide an option to allow defining the amount of time, but this should be enough to catch main issues.

  • ghvmctl: Try prevent gnome-initial-setup to start

Configure the machine so that initial setup won’t start.

Sadly it’s not possible to do this by configuring the image earlier, as trying to mount files while configuring may lead to permission errors.

So just do it as early as possible, keeping a fallback to still stop the app if this is required

  • ghvmctl: Disable lock screen and idle screen on VM

After a while the VM screen may become locked or become black, we need to prevent this to happen.

Using lockdown gsettings files would be better, but this would involve a custom image, given that mounting files doesn’t seem to work as expected.

  • ghvmctl-runner: Duplicate env variables from current user session

Instead of guessing env variables, we should inherit them from current user session, this can be done getting them from systemd (or checking the environ of an already known process for the uer), so that we can safely run in a very similar environment.

This could also be done using systemd-run, although it may have other implications too.

  • ghvmctl-runner: Use systemd-run to exec in user env

We can just run a process inside the session with all the right environment variables, leaving this job to systemd without having to do extra work.

This seems to do the right job after some testing, so let’s just simplify everything.

Keeping the old commit (instead of squashing), just in case we need to revert to that one day.

DateDec 14, 2023

feat: enable check for snap running, or to stop/kill snaps (#3)

  • ghvmctl: Deprecate -snap commands and use snap- instead

As it happens with screenshots, it’s more natural to have subcommands that have the prefix of the action that is going to be taken.

So snap-install, snap-run are better than the install-snap or run-snap.

At the same time add commands to stop or kill a snap, by using systemd so that we don’t have to deal on all the subprocesses around.
